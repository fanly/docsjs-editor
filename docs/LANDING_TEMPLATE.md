# Plugin Landing Template (docsjs Ecosystem)

Use one shared information architecture for all docsjs plugin pages.

## Canonical Section Order

1. `nav`
2. `hero`
3. `features`
4. `showcase`
5. `api-section`
6. `release`
7. `sponsors`
8. `footer`

## Stable Contract

- Keep section order and anchor IDs (`#features`, `#showcase`, `#api`)
- Keep bilingual i18n (`en` and `zh`)
- Keep reciprocal cross-promo block
- Do not hardcode version numbers in recommendation copy

## Theme Policy

Theme can vary by plugin:

- colors
- fonts
- gradients/textures
- card style

Structure must stay consistent for ecosystem scalability.
