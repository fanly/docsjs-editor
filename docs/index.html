Why docsjs-editor?

Mainstream Coverage

Built-in adapters for Tiptap, Quill, CKEditor5, TinyMCE, Toast UI, WangEditor and extensible hooks for Lexical/Slate/ProseMirror.

Switchboard Runtime

Switch editor providers in runtime while preserving current content and keeping business APIs stable.

docsjs Native Bridge

Bind docsjs events directly and pipe HTML snapshots into your target editor in one step.

Ecosystem Pairing

Combine with docsjs and docsjs-markdown to build full Word -> Editor -> Markdown workflows.

Demo & Matrix Ready

Use multi-switch demo and capability matrix to evaluate editor fit before production rollout.

Complete Case: Left docsjs, Right active editor

import { EditorSwitchboard, bindDocsjsChangeToEditor } from "@coding01/docsjs-editor";

const switchboard = new EditorSwitchboard();
const adapter = switchboard.connect("tiptap", tiptapEditor);
bindDocsjsChangeToEditor(docsjsElement, adapter);

await switchboard.switchTo("quill", quillEditor);
